# Tâche 6 — (Option) Protéger `main` sur GitLab

**CHRONO N°6** | Branche : — (réglage GitLab) | Qui : **M3**

## Ce que vous devez faire

1. Sur **GitLab UFR**, ouvrir le dépôt du projet → **Settings** → **Repository** (ou **Protected branches** selon la version).
2. Protéger la branche **`main`** :
   - interdire les push directs si le sujet le recommande ;
   - n’autoriser les fusions que via **merge request** depuis `develop` (ou équipe mainteneur uniquement).

Les libellés exacts dépendent de l’interface GitLab de votre établissement.

## Vérification

- Les membres du trinôme ne peuvent plus pousser sur `main` par erreur (sauf rôles autorisés).
